# Fullstack App — Node.js + React

Серверное приложение на Node.js/Express с JWT-аутентификацией, системой ролей и React-фронтендом.

## Проделанная работа

**ПР7 — Базовый REST API**
Создано серверное приложение на Node.js. Реализованы маршруты регистрации и входа, полный CRUD для товаров. Пароли хешируются через bcrypt, идентификаторы генерируются через nanoid.

**ПР8 — JWT аутентификация**
При входе сервер выдаёт access-токен. Добавлен authMiddleware для проверки Bearer-токена в заголовке Authorization. Маршруты GET/PUT/DELETE /api/products/:id и GET /api/auth/me защищены — доступны только авторизованным пользователям.

**ПР9 — Refresh токены**
При входе выдаётся пара токенов: access (15 минут) и refresh (7 дней). Добавлен маршрут POST /api/auth/refresh, который принимает refresh-токен и возвращает новую пару. Реализована ротация: старый refresh-токен при обновлении инвалидируется.

**ПР10 — React фронтенд**
Реализована клиентская часть на React с маршрутизацией через React Router. Страницы: вход, регистрация, управление товарами. Токен хранится в localStorage и подставляется в каждый запрос. Реализован полный CRUD для товаров через UI.

**ПР11–12 — Система ролей**
Добавлена система ролей: guest, user, seller, admin. Роль хранится в модели пользователя и включается в JWT. Реализован roleMiddleware для проверки прав на каждом маршруте. Новые пользователи получают роль user при регистрации. Добавлены маршруты управления пользователями (только для admin): список, редактирование, блокировка. Фронтенд адаптирован под роли: форма создания/редактирования товара видна только seller и admin, кнопка удаления — только admin, вкладка «Пользователи» — только admin.

## Стек

- **Backend:** Node.js, Express, JWT (jsonwebtoken), bcrypt, nanoid
- **Frontend:** React, React Router

## Запуск

**Backend**
```bash
npm install
node server.js
```
Сервер: `http://localhost:3000`

**Frontend**
```bash
npm install
npm run dev
```

## Роли

- **Гость** — регистрация, вход, обновление токенов
- **Пользователь** — просмотр товаров
- **Продавец** — создание и редактирование товаров
- **Администратор** — удаление товаров + управление пользователями

## API

| Маршрут | Метод | Доступ |
|---|---|---|
| /api/auth/register | POST | Гость |
| /api/auth/login | POST | Гость |
| /api/auth/refresh | POST | Гость |
| /api/auth/me | GET | Пользователь |
| /api/users | GET | Администратор |
| /api/users/:id | GET | Администратор |
| /api/users/:id | PUT | Администратор |
| /api/users/:id | DELETE | Администратор |
| /api/products | POST | Продавец |
| /api/products | GET | Пользователь |
| /api/products/:id | GET | Пользователь |
| /api/products/:id | PUT | Продавец |
| /api/products/:id | DELETE | Администратор |

> DELETE /api/users/:id не удаляет пользователя, а блокирует его (поле `blocked: true`).
