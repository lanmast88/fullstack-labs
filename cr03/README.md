# Веб-приложение для управления заметками (PWA)

Прогрессивное веб-приложение (PWA) для управления заметками с поддержкой офлайн-режима, push-уведомлений и напоминаний. Выполнено в рамках практических занятий №13–17.

## Практические задания

### Практика №13 — Service Worker и офлайн-режим

- Реализовано базовое приложение для управления заметками
- Данные сохраняются в `localStorage`
- Зарегистрирован Service Worker, кэширующий статические ресурсы
- При отсутствии сети страница загружается из кэша, просмотр и добавление заметок работают офлайн

### Практика №14 — Web App Manifest

- Создан `manifest.json` с полями: `name`, `short_name`, `start_url`, `display`, `background_color`, `theme_color`, `description`, `icons`
- Подготовлен набор иконок разных размеров (16×16 — 512×512) в формате PNG
- Манифест подключён в HTML через `<link rel="manifest">`
- Добавлены мета-теги для улучшения интеграции с мобильными платформами
- Service Worker обновлён для кэширования иконок и манифеста
- Приложение устанавливается на компьютер и открывается в отдельном окне

### Практика №15 — App Shell и стратегии кэширования

- Реализована архитектура App Shell
- Приложение работает по HTTPS (самоподписанный сертификат для разработки)
- При первом посещении кэшируются все статические ресурсы
- Динамические страницы загружаются по стратегии Network First
- Добавлена страница «О нас» (`/content/about.html`), доступная офлайн

### Практика №16 — Socket.IO и push-уведомления

- Реализована серверная часть на Express + Socket.io
- Сгенерированы VAPID-ключи для Web Push
- Клиент подключается к серверу через Socket.io:
  - при добавлении заметки отправляется событие `newTask`
  - при получении события `taskAdded` от сервера отображается всплывающее сообщение
- Реализованы функции `subscribeToPush` и `unsubscribeFromPush` через `PushManager`
- Добавлены кнопки включения/отключения уведомлений
- Подписка отправляется на сервер (`POST /subscribe`), удаляется при отписке (`POST /unsubscribe`)

### Практика №17 — Напоминания

- Добавлена форма создания заметки с напоминанием (текст + дата/время)
- Заметки в `localStorage` содержат уникальный идентификатор и поле `reminder` (timestamp)
- На сервере реализовано планирование push-уведомлений через `setTimeout` с хранением активных таймеров
- В Service Worker добавлена обработка действий уведомлений: кнопка «Отложить на 5 минут» и запрос к серверу (`POST /snooze`) для переноса напоминания
- Напоминания работают даже при закрытом приложении

## Технологии

**Frontend:**
- Vanilla JavaScript
- HTML5 + CSS3
- Chota CSS (фреймворк)
- Service Workers API
- Push Notification API
- localStorage

**Backend:**
- Node.js + Express
- Socket.io
- web-push

## Структура проекта

```
cr03/
├── index.html          # Главная HTML-страница
├── app.js              # Клиентская логика
├── sw.js               # Service Worker
├── style.css           # Стили
├── manifest.json       # Web App Manifest
├── content/
│   ├── home.html       # Содержимое главной страницы
│   └── about.html      # Страница «О нас»
├── icons/              # Иконки PWA (16×16 — 512×512)
└── server/
    └── server.js       # Express-сервер с Socket.io и Web Push
```

## Запуск

```bash
npm install
node server/server.js
```

Приложение доступно по адресу `https://localhost:3001`.

> Для работы push-уведомлений требуется HTTPS. В режиме разработки используются самоподписанные сертификаты (файлы `localhost+2.pem` и `localhost+2-key.pem`).
